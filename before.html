A common problem: corruption through common handling methods.

A common problem is that the very act of sending files through email can corrupt them. It's important to know that in order to enable an end user to be able to address the corruption, normally, permissions have to be set properly. Sometimes, simply routing a file through a "reader" version of sofware is enough to corrupt it. See the Discussion Forum, "Reader DC Send File corrupts additional attached file" for an example of this. There's also common issues with proprietary systems that do not always play well together. In many cases, support requires a particular, proprietary setup in order to even start. A common example of this is in opening proprietary Adobe products in a Microsoft environment.

If you want to manually set component security settings back to the default settings, follow these steps:

  1. Click Start, type dcomcnfg in the Search box, and then click dcomcnfg in the Programs list. See Figure 1 below from the Microsoft article.
